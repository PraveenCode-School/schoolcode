# 🔔 Notification & Login Persistence Fix Plan

## 📋 Current Issues

### Issue 1: Notifications Only Work When User is Logged In
**Problem:** `NotificationRegistration.jsx` only sets up listeners when `user` exists.
**Effect:** When app is swiped away or closed, notifications stop working.

### Issue 2: Auto-Logout When App is Cleared
**Problem:** Android might be killing the app state when cleared from recents.
**Effect:** Users get logged out when they swipe app away.

### Issue 3: Notifications Don't Persist
**Problem:** Foreground notifications show as toasts, not persistent system notifications.
**Effect:** Notifications disappear after a few seconds.

---

## ✅ SOLUTION APPROACH

### Fix 1: Move Notification Registration to App Level (OUTSIDE AuthContext)

**Change:**
```javascript
// CURRENT: NotificationRegistration.jsx
useEffect(() => {
    if (!user || !Capacitor.isNativePlatform()) return; // ❌ Requires login!
    setupNotifications();
}, [user]);

// NEW: App.jsx
useEffect(() => {
    if (!Capacitor.isNativePlatform()) return;
    setupNotifications(); // ✅ Always setup, regardless of login
}, []); // Empty dependency - runs once on app start
```

**Why:** FCM token registration should happen REGARDLESS of login state.
When notification arrives, it will be displayed by Android OS automatically.

---

### Fix 2: Register FCM Token After Login

**Flow:**
1. App starts → Request notification permission → Get FCM token
2. User logs in → Send FCM token to backend immediately
3. Backend stores token in `users.fcm_token` column
4. User logs out → Token stays registered (for background notifications)

**Changes:**
- Keep token in device storage even after logout
- Send token to backend after every login
- Backend can send notifications even if app is closed

---

### Fix 3: Make Notifications Persistent (WhatsApp-style)

**Android Configuration:**
```javascript
// Create PERSISTENT notification channel
await PushNotifications.createChannel({
    id: 'school_notifications',
    name: 'School Notifications',
    importance: 5, // MAX importance
    visibility: 1, // PUBLIC
    vibration: true,
    sound: 'default', // Add sound
    lights: true, // LED indicator
    lightColor: '#0EA5E9', // Blue light
});
```

**Notification Data Structure (Backend):**
```javascript
// Firebase payload
{
    notification: {
        title: "Attendance Update",
        body: "Your ward reached school at 9:30 AM",
        sound: "default",
        badge: "1",
        tag: "attendance", // Same tag = replaces old notification
    },
    android: {
        priority: "high",
        notification: {
            channelId: "school_notifications",
            sticky: true, // Can't be swiped away easily
            ongoing: false, // Not a permanent notification (allow dismissal)
            autoCancel: false, // Don't auto-dismiss when clicked
            showTimestamp: true,
        }
    },
    data: {
        type: "attendance",
        timestamp: "2026-02-03T09:30:00Z"
    }
}
```

## ⚠️ IMPORTANT NOTES

1. **FCM Token Lifetime:** FCM tokens can expire or change. Always re-register on app start.

2. **Android 13+ Permission:** Must request `POST_NOTIFICATIONS` permission (already added in manifest ✅)

3. **Background Restrictions:** Some manufacturers (Xiaomi, Huawei) kill apps aggressively. 
   Users might need to whitelist the app in battery settings.

4. **Testing:** Use real device, not emulator, for accurate notification behavior.

5. **Firebase Project:** Ensure `google-services.json` is up-to-date and matches your Firebase project.
